## Context
The audience is an experienced-but-rusty engineer. Passive reading doesn't
rebuild proficiency; they need to write code against a precise, checkable
target without being handed the answer.

## Decision
Every function-building exercise ships as: scaffold + `...` placeholder +
a technique-naming hint + asserts. The asserts are the executable spec —
silence when run means done. Refactor (rewrite-this) exercises are the only
assert-exempt type.

## Alternatives considered
- Worked examples with explanations — rejected: reading solutions doesn't
  rebuild recall (see DEC-0002).
- Exercises with separate answer keys — rejected: leaks solutions into the
  repo and invites peeking; the asserts already define correctness.

## Consequences
Authoring cost is front-loaded into writing good asserts (each part needs a
verification pass, DEC-0005). Enforced day-to-day by RUL-0001 and
tools/lint_workbook.py.
