## Context
Part 0 teaches Modern Python Project Structure. Teaching it from a fake,
non-runnable example would let errors ship unnoticed and give the learner
nothing to poke at.

## Decision
The workbook directory is itself a real project: `pyproject.toml` (hatchling
backend, `requires-python >= 3.11`), `src/crash_course/` stub package, and a
`dev` extra installing ruff, mypy, and pytest. `ruff check .`, `mypy src`,
and `pytest` must always run cleanly against it.

## Alternatives considered
- Illustrative snippets only — rejected: unverifiable, and Part 0 is the
  reference that chapters 14/15/18 drill against.
- Poetry / setuptools — rejected as default: hatchling is the leaner modern
  choice for a pure-Python src layout; nothing here needs more.

## Consequences
The toolchain doubles as a regression check on the repo itself. Chapters
14/15/18 reference this structure and must never re-define it.
