## Context
Verifying part-0 (2026-07-02) revealed that bare `pytest` exits **5** ("no tests
collected") on a project with no test files. That broke Part 0's claims as
written: exercise 0.4a's `ruff check . && mypy src && pytest` chain and
exercise 0.4b's pre-commit pytest hook could never pass, and DEC-0006's
"pytest must stay green" was silently unmet.

## Decision
Ship `python-crash-course/tests/test_smoke.py` — one test importing
`crash_course` and asserting `__version__ == "0.1.0"`. The 0.1 tree in
part-0.md includes `tests/` with a grounding comment stating the exit-5 gotcha.

## Alternatives considered
- No tests, weaken part-0's wording to avoid bare `pytest` — rejected: teaches
  a project shape where the pytest gate is configured but meaningless.
- pytest config to tolerate an empty suite — rejected: no clean stdlib/pytest
  option; masks the real signal.

## Consequences
`pytest` is a truthful gate everywhere it appears (Part 0 verify-its,
pre-commit hook, RUL-0002 toolchain checks). The smoke test doubles as a
version-drift tripwire (`__version__` vs pyproject is exercised in 0.5).
